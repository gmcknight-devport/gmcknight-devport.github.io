    • Project Details

    Consists of 5 microservices:

      Account microservice - Controls all authentication and user information in the project. Implements CQRS, Mediatr, communicates with Service Bus, EF Core persistence.
      Posts microservice - Responsible for all posts on the platform and the features within those - comments, likes, comment likes, and deletion. Implements CQRS, Mediatr, communicates with Service Bus, EF Core persistence.
      Media microservice - Controls the media used in the project. Currently only allows images but potential for video in the future. Communicates directly with Azure Storage Account. Only accessed through the aggregator or communicated with using integration events.
      Aggregator microservice - Links microservice together using API calls - retrieves user profile data, retrieves feed data, creates posts (requires Account, Posts, and Media microservices), and searches Account and Posts microservices using fuzzy search interface implemented in each.
      Gateway microservice - Ocelot used for testing the APIs before deployment on AKS.

    Kubernetes is used for the ingress controller, Secrets Store CSI driver and AKV2K8S to control access and security, and deployment of the pods for each microservice.

    Frontend is written in React and CSS, communicates with the microservices through API calls and is designed to be responsive for browser and mobile.

    Features

  • Keras models - LSTM, Recurrent Neural Network, Convolutional Neural Network, GRU, BiDirectional Model, Random Forest
  • Sentiment Analysis and web scraping
  • Ensemble models combining ML and Sentiment Analysis - allows any combination Keras model with Sentiment Analysis

    • Project Details

    Research Project - used a series of clients allowing factories for different types of ML models to be implemented and analysed for accuracy and efficacy.

    FastAPI implementation of selected models allowing a frontend to be built for a user to experiment with the models.

    Streamlit implementation allows users to interact with the models.

    Though the research was extensive, due to the lack of predictable markers, the models resort to following a random walk and copying the previous data point resulting in a graph and accuracy score that look almost perfect, belying the fact that the algorithm is always 1 step behind the market. The various implementations could not outperform a buy-and-hold strategy due to the implications of the Random Walk and Efficient Market hypothesis. Models with a basis in traditional financial analysis provided a much better end result, usually outperforming the baseline, providing solid and reliable information to make executive decisions.

    There is potential for Machine Learning tools to be implemented to support traditional stock forecasting tools and industry professionals alongside Sentiment Analysis to more accurately predict market movements.
